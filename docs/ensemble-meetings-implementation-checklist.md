# Ensemble Meetings Implementation Checklist

This checklist turns the ensemble meetings design note into a concrete first coding pass.

## Target slice

Implement an active-scene ensemble summary panel in meetings view when the current scene has three or more characters.

## Inputs to derive

From the active scene:

- scene id
- scene title
- optional time label
- optional location id and resolved location name
- ordered cast ids
- resolved cast records

From the cast:

- all distinct in-scene pairs
- per-character pair count inside the active scene
- total pair count

From the current meetings dataset:

- which visible pair cards are generated by the active scene
- whether those active-scene pairs need sorting or highlighting treatment

## First implementation steps

1. Find the meetings-view render path and isolate where active-scene arrival context is currently produced.
2. Add a pure helper that derives ensemble metadata from the active scene and character records.
3. Return `null` from that helper when cast size is below three.
4. Render a dedicated ensemble summary card above the pair list when helper output exists.
5. Add a visual marker on pair rows generated by the active ensemble.
6. Reuse the existing jump-back-to-scene-editor action from the current meetings context.
7. Verify the layout in both light and dark themes.

## UI contents for v1

The summary card should contain:

- scene title
- optional time label
- optional location name
- cast list in scene order
- total cast count
- total distinct pair count
- short "most connected" line based on in-scene pair counts
- button or link back to scene editing

## Deliberately excluded from v1

- graph visualisation
- persistent relationship data
- historical clustering across multiple scenes
- editable relationship labels
- speculative social inference beyond explicit scene co-presence

## Acceptance test

Using a demo scene with five present characters:

- the ensemble summary card appears above the meetings list
- the card names the scene and its context cleanly
- the cast list is readable at a glance
- pair count matches combinatorics for the cast size
- pair rows tied to the active scene are visually distinguishable
- the user can jump back to the current scene editor without losing context

## Good follow-up if v1 lands cleanly

- sort active-scene pairs to the top temporarily
- add compact badges for active-scene rows
- show whether a pair appears only in this scene or across many scenes
